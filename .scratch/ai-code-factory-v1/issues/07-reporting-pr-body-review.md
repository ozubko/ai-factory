# Full reporting — report.md + pr-body.md + optional Diff Review

Status: done

## Parent

PRD: `.scratch/ai-code-factory-v1/PRD.md`

## What to build

A complete `report.md` that leads with the Risk Assessment and the
factory-verified-vs-agent-claimed distinction, states whether the Factory continued
automatically or paused (and why), summarizes changed files + verification results,
and ends with concrete human next-steps (inspect worktree, switch branch). A
forge-neutral `pr-body.md` (Summary / What Changed / Why / Verification / Test
Coverage Notes / Risks & Follow-ups / Human Review Notes) with a "generated by
ai-factory; verify before merging" footer. An optional `--review` Diff Review Phase
(read-only) that critiques the Run's diff and feeds findings into the report. The
Factory still never merges, pushes, or opens a PR. See ADR-0003, 0005, 0011, 0014.

## Acceptance criteria

- [ ] `report.md` leads with Risk Assessment + factory-vs-agent-claims and states continued/paused + why
- [ ] `pr-body.md` is forge-neutral and includes the verify-before-merging footer
- [ ] `--review` runs a read-only Diff Review whose findings appear in the report
- [ ] No merge/push/PR/remote operation occurs in any path

## Comments

Implemented. `report.md` already led with Risk Assessment and the
factory-verified-vs-agent-claimed distinction from issue 06; this issue added:

- **`pr-body.md`** (`src/ai_factory/pr_body.py`, `render_pr_body`): forge-neutral
  (no GitHub/GitLab-specific syntax), written for every Run alongside
  `report.md`, with `Summary / What Changed / Why / Verification / Test Coverage
  Notes / Risks & Follow-ups / Human Review Notes` sections and a
  `_Generated by ai-factory; verify before merging._` footer.
- **`--review` Diff Review Phase** (opt-in, `runner.py`): a read-only Phase run
  after the Fix Loop, over the Run's diff, using the existing `review` prompt
  (`prompts.py`) and Fake Agent behavior that already existed from earlier
  planning. Enforced via git the same way as `plan` -- a mutation is a Contract
  Violation with saved evidence (`contract-violation.patch` /
  `-files.txt`) and halts the Run. Findings land in `metadata.json`
  (`phases.review`) and in `report.md`'s new `## Diff Review` section and
  `pr-body.md`'s `Risks & Follow-ups`. It never gates the Run's outcome (not an
  approval gate) except for its own Contract Violation. Defaults to
  `status: "skipped"` when `--review` is not passed, and `"not_executed"` when
  passed but the Run never reached a completed implementation (paused, halted,
  or implement itself failed).
- Added `--mutate-readonly-phase` to the Fake Agent and a `fake-review-violator`
  preset so the review Contract Violation path could be exercised in isolation
  from the `plan` Phase's own Contract Violation path (both share
  `--mutate-readonly`).
- No push/merge/PR/remote code path was added anywhere; confirmed via a new
  end-to-end test and manual CLI run that the target repo's working tree,
  branches, and remotes are untouched.

Tests: `tests/test_reporting_pr_body_review.py` (13 new tests, all passing) plus
the full suite (96 passed), `ruff`, and `mypy` clean. Manually drove
`ai-factory run ... --review` against a real temp git repo and inspected the
